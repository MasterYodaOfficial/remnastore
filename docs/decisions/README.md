# Архитектурные решения

Здесь хранятся ADR (Architecture Decision Records) по важным техническим решениям проекта.

Примеры того, что стоит фиксировать:
- почему выбран конкретный способ аутентификации
- как устроен ledger и почему он append-only
- как интегрируются платежные провайдеры
- почему админка вынесена в отдельное приложение

Если решение влияет на архитектуру, безопасность, данные или операционные процессы, его лучше зафиксировать отдельным ADR.
