# Release Notes

В этой директории храним финальные release notes для merge `dev -> main`.

Правила:

- один релиз = один файл `v0.x.y.md`
- перед первым релизом используем [`TEMPLATE.md`](./TEMPLATE.md)
- после публикации GitHub Release текст в UI не должен расходиться с файлом в репозитории

Примеры имен:

- `v0.1.0.md`
- `v0.1.1.md`
- `v0.2.0.md`
